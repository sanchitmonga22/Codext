Build a polished, single-page landing site for Codext at landing/index.html. The page should be pure HTML + inline CSS in a single file — no build tools, no JavaScript frameworks, no external CDN dependencies, no tracking scripts. The page must open and look correct when double-clicked from Finder, with no dev server required.

VISUAL IDENTITY
- Background: vertical linear gradient from #B1AAF8 (lavender, top) to #444AF5 (electric blue, bottom). This matches assets/logo/codext-logo.svg.
- Hero shows the actual logo from assets/logo/codext-logo.svg (reference it as ../assets/logo/codext-logo.svg, or inline the SVG).
- Typography: Inter or system-ui font stack. White text for headings on the gradient, near-white for body copy.
- Content sections use white rounded-corner cards (12-16px radius) with a subtle shadow for depth.
- Generous whitespace, calm modern feel.

SECTIONS
1. Hero
   - The Codext logo
   - The "Codext" wordmark
   - Tagline: "Codex, in a Loopt — autonomous PR loops while you sleep"
   - A copy-able install command (the curl one-liner from README.md), shown in a monospace pill with a copy hint

2. Three feature cards in a grid (3 columns on desktop, 1 column under 720px):
   - "Iterative loop" — branch-per-iteration PRs with CI checks and auto-merge.
   - "Budget caps" — control runs by --max-runs, --max-tokens, or --max-duration.
   - "Model tuning" — pick a model with --model, dial reasoning with --effort, or accelerate with --fast.

3. CTA section
   - A "View on GitHub" button linking to https://github.com/sanchitmonga22/Codext
   - Small footer with the MIT license note and a credits line for continuous-claude

CONSTRAINTS
- Single file at landing/index.html. Inline <style> for CSS. No external assets except the local SVG logo.
- Mobile-responsive: single column under 720px viewport.
- Semantic HTML (use <header>, <main>, <section>, <nav>, <footer> where appropriate).
- Accessible: aria labels on links/buttons, alt text on the logo, sufficient color contrast (4.5:1 for body text), visible :focus styles.
- No <script> tags. Pure HTML + CSS only.

ITERATIVE PROGRESS GUIDANCE
This task is being built across multiple iterations. Each iteration should make a clearly visible improvement.
- Iteration 1: build the structure and basic styling — semantic HTML, gradient background, working sections.
- Iteration 2: polish — typography rhythm, spacing scale, card shadows, hover states, copy refinement.
- Iteration 3: final pass — responsive refinements, focus styles, accessibility audit, micro-interactions (subtle CSS transitions on hover).

Update SHARED_TASK_NOTES.md at the end of each iteration with what's done and what remains so the next iteration picks up cleanly.
